# 远程服务创建研判

## 目标
判断通过PSExec、SC等方式创建远程服务的告警是真实攻击，还是误报。

## 范围
- 远程服务创建
- PSExec执行
- SC命令远程执行
- WMI服务创建
- 横向移动
- 远程命令执行

## 输入
| 参数 | 说明 |
|------|------|
| `${alert:name}` | 告警名称 |
| `${alert:severity}` | 告警等级 |
| `${alert:srcHost}` | 源主机名 |
| `${alert:dstHost}` | 目标主机名 |
| `${alert:serviceName}` | 服务名称 |
| `${alert:servicePath}` | 服务执行路径 |
| `${alert:commandLine}` | 命令行参数 |
| `${alert:user}` | 执行用户 |

## 输出
| 参数 | 说明 |
|------|------|
| `${alert:triageResult}` | 研判结果 |
| `${alert:triageCoT}` | 研判思维链 |

## 工具
- **EDR工具**: `${edr-mcp}`
  - `${edr-getProcessTree}` - 获取进程树
  - `${edr-getFileHash}` - 获取文件哈希
  - `${edr-getFileContent}` - 获取文件内容
- **SOC工具**: `${soc-mcp}`
  - `${soc-getAlert}` - 获取告警信息
  - `${soc-getAsset}` - 获取资产信息
  - `${soc-getTcpLog}` - 获取TCP日志

## 流程

### 正报判断

#### 1. 服务特征分析
- **1-1.** 分析服务名称
  - 从 `${alert:serviceName}` 识别可疑服务名
  - 检查是否为随机字符串（如 `kesknq`、`abcdef`）
  - 识别是否为伪装名称（模仿系统服务但路径异常）
  - 检查是否为攻击工具特征名

- **1-2.** 分析服务路径
  - 从 `${alert:servicePath}` 识别可疑路径
  - 检查是否在临时目录：`%TEMP%`、`C:\Windows\Temp\`
  - 识别是否为UNC路径：`\\127.0.0.1\ADMIN$\`
  - 检查是否指向随机命名的可执行文件

- **1-3.** 分析命令行
  - 从 `${alert:commandLine}` 识别攻击特征
  - 检查是否包含PowerShell混淆代码
  - 识别是否包含Base64编码命令
  - 检查是否为Metasploit风格的载荷执行

#### 2. 横向移动分析
- **2-1.** 分析移动路径
  - 检查 `${alert:srcHost}` 和 `${alert:dstHost}` 的关系
  - 判断是否从普通主机向关键服务器移动
  - 识别是否向域控制器创建远程服务

- **2-2.** 分析执行用户
  - 从 `${alert:user}` 判断账户类型
  - 检查是否使用域管理员账户
  - 识别是否为服务账户（可能已被转储）
  - 检查是否为异常账户使用

#### 3. 关联行为分析
- **3-1.** 分析前置行为
  - 通过 `${soc-getAlert}` 获取源主机近24小时告警
  - 检查是否存在LSASS凭证转储告警
  - 识别是否存在权限提升告警

- **3-2.** 分析目标主机后续行为
  - 通过 `${soc-getAlert}` 获取目标主机后续告警
  - 检查服务创建后是否有可疑进程执行
  - 识别是否发起新的横向移动

- **3-3.** 分析网络流量
  - 通过 `${soc-getTcpLog}` 获取SMB/RDP流量
  - 检查是否存在多主机快速切换的RDP会话
  - 识别SMB文件共享的异常使用

### 误报判断

#### 1. 合法管理分析
- **1-1.** 分析管理工具
  - 检查源主机是否为IT管理服务器
  - 识别是否为已知的系统管理工具（SCCM、Altiris等）
  - 判断是否为合法的远程管理软件

- **1-2.** 分析服务特征
  - 检查服务路径是否为已知的管理代理
  - 识别是否为公司标准的监控或运维工具
  - 判断服务名称是否符合命名规范

#### 2. 资产属性分析
- **2-1.** 分析资产关系
  - 通过 `${soc-getAsset}` 获取源和目标主机信息
  - 判断是否为同一业务系统内的主机
  - 检查是否存在正常的管理关系

#### 3. 历史行为分析
- **3-1.** 分析历史记录
  - 通过 `${soc-getAlert}` 获取历史远程服务告警
  - 判断是否存在相同源/目标的重复行为
  - 检查是否在维护窗口时间内
